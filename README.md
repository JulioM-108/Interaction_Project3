# Interaction Systems - Project 3
This repository contains all the work developed for the "Sistemas de Interacción" Project 3. The objective was to create an algorithm in OpenMusic to generate a musical piece for a video game.

## Objective:
To develop an algorithm using OpenMusic (OM) that generates an original musical piece for a selected video game. The final composition must be cohesive with the game's atmosphere and adhere to all technical project constraints. 


## Project Constraints:
To complete the project, several technical and creative limitations were established:

* Software: The project must be developed exclusively in OpenMusic. 

* Original Music: No pre-existing musical material (such as WAV, MIDI, or MP3 files) could be used. All music had to be generated by the algorithm. 

* Duration: The final musical piece must be at least two minutes long. 

* Originality: The piece must contain at least one minute of non-repetitive, or "fresh," music. 

* Final Delivery: The submission includes a public GitHub repository with the source code and this README, and a YouTube video showing gameplay with the generated music. 


## Proposed Solution:
For this project, we chose gameplay from the upcoming title **Silent Hill F**. This game is described as a psychological horror experience set in 1960s Japan, focusing on unsettling and atmospheric horror rather than simple jump scares.

Our goal was to create a musical piece that enhances this specific mood. We aimed to generate a soundtrack that conveys a sense of creeping dread, tension, and psychological unease, fitting the game's visual and thematic style.

To achieve this in OpenMusic, our algorithm was built using several core concepts:

We used random number generators constrained to specific dissonant scales and modal harmonies to create an unpredictable yet musically cohesive and unsettling melodic texture.

Data structures like lists were heavily used to manage musical parameters such as pitch, duration, and velocity. We manipulated these lists with functions like concat, append and division to construct the final composition algorithmically.

## Conclusion:
We believe we successfully met the project's objective. The generated music creates an eerie and suspenseful atmosphere that complements the psychological horror of Silent Hill F. The piece fulfills all technical requirements, including the two-minute total duration and the one-minute "fresh music" rule.  The use of algorithmic generation in OpenMusic allowed us to create a unique soundscape tailored specifically to the chosen gameplay.
